# Safe Workflow

`main` в этом репозитории считается стабильной версией.

## Правило по умолчанию

Перед любыми заметными изменениями:

1. создать рабочую ветку от текущего `main`;
2. вносить правки только в этой ветке;
3. после проверки либо влить ветку в `main`, либо удалить её.

## Быстрый старт

Из корня проекта:

```bash
./scripts/start-change.sh hero-fix
```

Скрипт:

- переключит репозиторий на свежую ветку вида `work/20260516-1600-hero-fix`;
- создаст локальный backup-tag вида `snapshot/20260516-1600-hero-fix`;
- попытается сразу опубликовать ветку в `origin`.

## Рабочий цикл

1. Создать ветку:

```bash
./scripts/start-change.sh short-task-name
```

2. Внести изменения.

3. Проверить результат.

4. Зафиксировать:

```bash
git add .
git commit -m "Describe the change"
git push
```

5. Если правки приняты, вернуть их в `main`:

```bash
git switch main
git pull --ff-only
git merge --ff-only work/<branch-name>
git push
```

## Как откатиться

Если новая версия не нужна:

- просто не вливать рабочую ветку в `main`;
- при необходимости вернуться к backup-точке:

```bash
git switch main
git reset --hard snapshot/<tag-name>
```

`git reset --hard` меняет файлы принудительно. Использовать только если точно нужен полный откат.
